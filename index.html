Core Java OOP for Interviews

This guide provides an interactive overview of Object-Oriented Programming (OOP) principles in Java, tailored for interview preparation. OOP is a programming paradigm based on the concept of "objects", which can contain data in the form of fields (often known as attributes or properties) and code in the form of procedures (often known as methods). This approach helps in structuring software in a way that is reusable, manageable, and scalable.

Classes & Objects

In OOP, a class is a blueprint for creating objects. It defines a set of properties and methods that are common to all objects of that type. An object is an instance of a class. Think of a `Car` class as the blueprint, while individual cars (a specific Ford Mustang, a specific Honda Civic) are the objects created from that blueprint.

The Four Pillars of OOP

OOP is built upon four fundamental principles, often called pillars. These concepts work together to create robust and flexible applications. Understanding each pillar and how they relate to one another is crucial for any Java developer.

1. Encapsulation

Bundling data (attributes) and methods that operate on the data into a single unit (a class), and restricting access to some of the object's components. This is a protective shield that prevents the data from being accidentally modified.

2. Inheritance

A mechanism where a new class derives properties and behaviors from an existing class. It promotes code reusability and establishes a relationship between the classes (parent-child or super-sub).

3. Polymorphism

The ability of an object to take on many forms. Most commonly, it allows a single action or method to be performed in different ways, depending on the object that is being acted upon.

4. Abstraction

Hiding complex implementation details and showing only the essential features of the object. It helps in managing complexity by focusing on "what" an object does instead of "how" it does it.

Pillar 1: Encapsulation

In Java, encapsulation is achieved by declaring the variables of a class as `private` and providing public `getter` and `setter` methods to modify and view the variable values. This prevents direct, uncontrolled access to the class's data fields.

public class BankAccount {
    // private variable - hidden from other classes
    private double balance;

    // Constructor
    public BankAccount(double initialBalance) {
        if (initialBalance >= 0) {
            this.balance = initialBalance;
        }
    }

    // public getter method - provides read-only access
    public double getBalance() {
        return this.balance;
    }

    // public setter method - provides controlled write access
    public void deposit(double amount) {
        if (amount > 0) {
            this.balance += amount;
        }
    }
    
    public void withdraw(double amount) {
        if (amount > 0 && amount <= this.balance) {
            this.balance -= amount;
        }
    }
}

Pillar 2: Inheritance

Inheritance allows a child class (subclass) to inherit the fields and methods of a parent class (superclass). Java uses the `extends` keyword. Java supports single and multi-level inheritance for classes but does not support multiple inheritance (a class cannot extend more than one class) to avoid complexity. Multiple inheritance of type is achieved using interfaces.

// Superclass (Parent)
class Animal {
    void eat() {
        System.out.println("This animal eats food.");
    }
}

// Subclass (Child) inherits from Animal
class Dog extends Animal {
    void bark() {
        System.out.println("The dog barks.");
    }
}

// Main class to run
public class TestInheritance {
    public static void main(String[] args) {
        Dog myDog = new Dog();
        myDog.eat();  // Inherited from Animal class
        myDog.bark(); // Own method
    }
}

Pillar 3: Polymorphism

Polymorphism means "many forms". In Java, it has two main types:

  • Compile-time Polymorphism (Method Overloading): Occurs when there are multiple methods with the same name but different parameters (number of arguments, type of arguments, or both) in the same class.
  • Run-time Polymorphism (Method Overriding): Occurs when a subclass has a method with the same name, parameters, and return type as a method in its superclass. The version of the method that gets executed is determined at runtime based on the object type.
// Method Overloading (Compile-time)
class Calculator {
    int add(int a, int b) {
        return a + b;
    }
    double add(double a, double b) {
        return a + b;
    }
}

// Method Overriding (Run-time)
class Vehicle {
    void run() {
        System.out.println("Vehicle is running");
    }
}

class Bicycle extends Vehicle {
    @Override // Annotation to indicate overriding
    void run() {
        System.out.println("Bicycle is running safely");
    }
}

public class TestPolymorphism {
    public static void main(String[] args) {
        Vehicle myVehicle = new Bicycle(); // Upcasting
        myVehicle.run(); // Calls the Bicycle's run() method at runtime
    }
}

Pillar 4: Abstraction

Abstraction hides implementation details and shows only functionality. In Java, it's achieved using abstract classes and interfaces.

Abstract Class vs. Interface: Key Differences

Feature Abstract Class Interface
Methods Can have both abstract and non-abstract (concrete) methods. Can only have abstract methods (Java 8+ allows default and static methods).
Variables Can have final, non-final, static, and non-static variables. Variables are `public static final` by default.
Inheritance A class can extend only one abstract class. A class can implement multiple interfaces.
Constructor Can have a constructor. Cannot have a constructor. 
// Abstract Class
abstract class Shape {
    abstract void draw(); // abstract method
    void display() {
        System.out.println("This is a shape.");
    }
}

class Circle extends Shape {
    void draw() {
        System.out.println("Drawing a circle.");
    }
}

// Interface
interface Drawable {
    void sketch(); // abstract method by default
}

class Rectangle implements Drawable {
    public void sketch() {
        System.out.println("Sketching a rectangle.");
    }
}

Other Key Concepts for Interviews

A constructor is a special method used to initialize objects. It is called when an object of a class is created. It must have the same name as its class and does not have an explicit return type. If you don't define a constructor, Java provides a default, no-argument constructor. Constructors can be overloaded (Constructor Overloading).

`this`: A reference variable that refers to the current object. It can be used to refer to the current class instance variable, to invoke the current class method, or to invoke the current class constructor.

`super`: A reference variable that is used to refer to the immediate parent class object. It can be used to access the parent class's instance variables, invoke the parent class's methods, or invoke the parent class's constructor.

`static`: Used for memory management. The `static` keyword can be applied to variables, methods, blocks, and nested classes. A static member belongs to the class itself rather than to an instance of the class.

`final`: Used to restrict the user. A final variable's value cannot be changed (it's a constant). A final method cannot be overridden. A final class cannot be inherited.

In C++, developers are responsible for manually deallocating memory using a destructor. In Java, memory management is automated. The Java Virtual Machine (JVM) has a process called the Garbage Collector that automatically identifies and deletes objects that are no longer being used by the program. This prevents common memory leak issues found in languages with manual memory management.
